Welcome to the super-cool-and-not-so-secret project archive documentation!

Here you can find simple documentation about the archive. I'm saying simple because it's really not that hard to understand.

Main endpoint

Yeah, it's here. Going to this site gives the current archived projects' directories in JSON. Here's an example.

{ "dirs": [ "142" ] }
While it may look a bit vague, it actually says a lot. Each directory is the name of a project ID. Which brings us to:

/archive/id endpoint

Using the project IDs, you can simply add it after the main endpoint and voila:

{ "files": [ "project142-0.sb3", "project142-1.sb3" ] }
You get another JSON response, which contains all the versions available for a project (that have been archived) . The number increment indicates how new the project is. The higher, the newer.
